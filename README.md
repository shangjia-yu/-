# 无线体感控制仿生机械臂

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

## 项目简介
本设计的核心目标是设计与实现一套基于Wi-Fi无线传输、体感控制的仿生机械臂系统。通过佩戴或手持集成了MPU6050姿态传感器的遥控终端，做出手腕左右倾斜（横滚角Roll）和手腕前后俯仰（俯仰角Pitch）两个动作，做到可远程控制机械臂的底座旋转和大臂俯仰，实现手部动作捕捉，机械臂同步跟随的仿生效果。具体目标分解如下：
1.	搭建一个完整的嵌入式系统，包括：
1)	遥控器：一个ESP8266开发板 + MPU6050传感器。拿着它，手腕倾斜时，它会通过Wi-Fi发出角度数据。
2)	执行端：另一个ESP8266开发板 + Arduino UNO+舵机。
2.	实现MPU6050姿态数据的实时采集与角度解算。
3.	利用UDP广播协议在局域网内无线传输角度数据，要求通信稳定、延迟低、丢包率小。
4.	设计舵机控制算法，包括中值滤波、死区抑制、速率限制和低通滤波，以消除手部抖动和电磁干扰，使机械臂运动平滑。
5.	最终能够运行：手持遥控端，分别做左右转腕和上下点头动作，机械臂底座和大臂随之同步转动，动作自然、无剧烈抖动。

## 基本原理

### 感知层（遥控端）
遥控端以 ESP8266-A（NodeMCU V3 开发板）为核心，外接 MPU6050 六轴惯性测量单元。系统上电后，ESP8266-A 首先通过 I2C 总线（SDA 接 GPIO4，SCL 接 GPIO5）初始化 MPU6050，将其从睡眠模式唤醒并配置为加速度计工作模式。在循环中，ESP8266-A 以约 20 Hz 的频率读取加速度计的三轴原始数据。利用三角关系计算出静态倾斜角度：
1)	俯仰角（pitch） = `atan2(AcY, AcZ) × 180/π`，范围为 -90° ~ 90°，正值表示手腕上扬。
2)	横滚角（roll）  = `atan2(AcX, AcZ) × 180/π`，范围为 -90° ~ 90°，正值表示手腕右倾。
这个算法仅依赖重力分量，在手腕缓慢运动时足够精确且计算量小。随后，ESP8266-A 将两个角度值格式化为字符串 "pitch,roll\n"。通过 UDP 协议向局域网广播：目标地址设置为广播地址 255.255.255.255，端口固定为 8888。采用广播模式无需预先知道接收端 IP，所有同一网络内绑定该端口的设备均可接收，简化了通信配置。每次发送后延时 50 ms，确保数据流平滑且不占用过多带宽。

### 传输层（中继端）
中继端由另一块 ESP8266-B 承担。它首先连接至与遥控端相同的 Wi-Fi 网络（必须为 2.4 GHz），然后创建一个 UDP socket 并绑定到端口 `8888`。在 `loop()` 中，程序调用 `udp.parsePacket()` 阻塞等待（非阻塞检查）来自网络的数据包。一旦接收到完整的 UDP 数据报，ESP8266-B 便读取其中的字符串内容（即 `"pitch,roll\n"`）。由于后续的 Arduino UNO 不具备网络功能，ESP8266-B 需将数据通过**串口**转发。具体采用硬件串口（`Serial`）或软串口（`SoftwareSerial`）取决于引脚分配。本设计中 ESP8266-B 的 TX 引脚（GPIO1）连接到 Arduino UNO 的软串口 RX 引脚（数字 12），波特率设为 115200，与 Arduino UNO 的软串口速率严格匹配。ESP8266-B 每收到一帧 UDP 数据，立即执行 `Serial.println(buffer)`，将原始字符串原样发送给 Arduino UNO。该中继方式既利用了 ESP8266 的 Wi-Fi 能力，又避免了更换 Arduino 主控的硬件成本，且数据传输延迟低于 10 ms，对实时控制无影响。

### 执行层（控制端）
1)	执行层核心为 Arduino UNO，负责解析中继端传来的角度数据并驱动舵机。具体流程分为五个子步骤：
2)	数字滤波：为抑制 MPU6050 的测量噪声及网络传输中的偶然抖动，Arduino 维护一个长度为 5 的滑动窗口，对连续 5 个 pitch 和 roll 值分别进行中值滤波。中值滤波能有效消除孤立的异常值，且保留边缘特征。滤波后的角度记为 `fPitch` 和 `fRoll`。
3)	死区判定：设定死区阈值 `DEAD_ZONE = 3°`。若当前角度与上一有效角度之差的绝对值小于该阈值，则认为手腕未移动，直接返回，不更新舵机。这一机制消除了手部生理性颤抖（通常幅度 1~2°）引起的舵机颤抖，同时减少舵机不必要的磨损。
4)	角度映射：将 `fRoll`（横滚角）线性映射到底座舵机的 PWM 输出角度 0~180°；将 `fPitch` 线性映射到大臂舵机的输出角度 30~150°（限制上下限可避免机械臂撞击自身）。映射采用 Arduino 的 `map()` 函数。
5)	速率限制：为防止传感器数值因快速甩动而产生突变（例如相邻两帧角度差超过 20°），设置最大每帧变化量 `MAX_RATE = 5°`。计算当前目标角度与上次实际输出角度的差值，若超出 ±5°，则仅变化 ±5°。这使舵机运动更加柔和，避免机械臂猛烈冲击。
6)	平滑输出：最后采用一阶低通滤波（指数平滑）：`smoothAngle = α × targetAngle + (1-α) × smoothAngle`，滤波系数 α 取 0.2。该滤波器进一步削弱高频抖动，使角度轨迹连续、自然。滤波后的角度值被写入舵机对象，通过 9 号引脚驱动底座舵机、6 号引脚驱动大臂舵机。
所有处理均在一次 `loop()` 周期内完成，实测单帧处理时间约 2~3 ms，远小于数据更新间隔 50 ms，系统具有充足余量。整个执行层不仅实现了角度到运动的映射，还集成了信号处理的经典方法，确保了机械臂的稳定性和跟随感。

### 姿态解算原理（基于加速度计的倾斜角计算）
MPU6050内部集成三轴加速度计和三轴陀螺仪。在设计中仅利用加速度计计算静态倾斜角，因为手腕动作频率较低（<5Hz），加速度计就足够了，且算法简单。
加速度计测量的是重力加速度在三轴上的分量。设传感器水平放置时，Z轴向上，重力加速度为g。当传感器绕X轴旋转角度θ（俯仰角pitch）时，Y、Z轴分量满足：tanθ = AcY / AcZ。同理，绕Y轴旋转角度φ（横滚角roll）满足：tanφ = AcX / AcZ。因此：
pitch = arctan2(AcY, AcZ) * 180 / π
roll  = arctan2(AcX, AcZ) * 180 / π
其中AcX、AcY、AcZ为从MPU6050读取的16位有符号整数（原始值）。该计算在ESP8266内完成，每秒约20次（delay(50ms)），满足实时性要求。
在这里需要特别强调的是，以上的设计只在动态加速度较小（手腕运动平缓）时有效。若快速甩动手腕，陀螺仪积分可提供更准确角度，但因为在设计中控制机械臂运动速度较慢，所以放弃。

### 无线通信原理（UDP广播）
UDP（用户数据报协议）是无连接的传输层协议，具有低开销、低延迟的特点，适合局域网内的实时数据广播。本设计中：
1.	遥控端创建UDP socket，目标地址设置为广播地址255.255.255.255，端口8888。所有同一局域网内的设备均可接收该报文。
2.	中继端绑定同一端口，调用udp.parsePacket()检查是否有新数据，读取后通过Serial.println()输出。
3.	优点：无需建立连接，一台遥控端可对多台中继端广播；缺点：不可靠，但局域网内丢包率极低，对控制影响可忽略。

### 舵机控制与抖动抑制算法
Arduino UNO接收到的pitch/roll可能存在噪声，原因包括：传感器量化误差、手部生理性颤抖、WiFi传输的偶然延迟抖动等。为了获得平滑的舵机运动，设计了四级处理流水线：
1.	中值滤波：维护一个长度为5的滑动窗口，对最近5次的角度值排序，取中位数作为有效值。该操作能有效滤除脉冲性噪声（如偶尔的异常读数）。
2.	死区抑制：设定死区阈值DEAD_ZONE = 3°。当当前角度与上一有效角度之差的绝对值小于该阈值时，认为手腕未移动，不更新舵机输出。此机制避免了手部微小颤抖引起的舵机颤抖。
3.	角度映射：将滤波后的roll（范围约-70°～70°）映射到舵机PWM对应的角度（底座：0～180°；大臂：30～150°）。使用Arduino的map()函数实现线性映射。
4.	速率限制 + 低通滤波：
1)	速率限制：限制每帧舵机角度的最大变化量MAX_RATE = 5°，防止因传感器瞬态跳变导致舵机猛转。
2)	一阶低通滤波（指数平滑）：smoothAngle = α × rawAngle + (1-α) × smoothAngle，其中α=0.2。该滤波器进一步平滑运动轨迹，使动作看起来更自然。
经过上述处理，舵机运动平滑、无可见抖动，且响应迅速。

### 供电与共地原理
舵机瞬间电流可达1A以上。如果直接从Arduino UNO的5V引脚取电，会拉低系统电压，导致单片机复位或舵机无力。因此必须使用外置5V/2A电源单独给舵机供电。同时，为了保证舵机控制信号正确（信号是相对于GND的电压），需要将外置电源的GND与Arduino的GND连接在一起，即“共地”。所有舵机的负极线、Arduino GND、ESP8266中继GND均应汇集到一点，避免形成地环路干扰。



## 接线说明

### 遥控端（ESP8266-A + MPU6050）
MPU6050 →  ESP8266 -A
VCC  → 3.3V 
GND →  GND 
SCL → D1 (GPIO5) 
SDA → D2 (GPIO4)

### 中继端（ESP8266-B<->Arduino UNO）
ESP8266-B -> Arduino UNO 
TX -> 数字引脚12 (RX)
RX -> 数字引脚13 (TX)
GND -> GND

### 舵机与电源
底座舵机信号线 -> 引脚9，大臂舵机信号线 → 引脚6
所有舵机红线 -> 外接5V电源正极
所有舵机棕/黑线 -> 外接电源负极 **并** 连接到Arduino GND（共地）

## 软件烧录步骤

1.安装ESP8266支持：Arduino IDE → 开发板管理器 → 搜索`esp8266` → 安装3.1.2。
2.烧录遥控端：打开`code/transmitter/transmitter.ino`，修改WiFi名称/密码，选择开发板`NodeMCU 1.0`，上传。
3.烧录中继端：打开`code/receiver/receiver.ino`，修改相同的WiFi信息，上传。
4.烧录Arduino UNO：打开`code/arduino_control/arduino_control.ino`，选择`Arduino Uno`，上传。

## 使用方法
1. 给所有设备通电（ESP8266-A、ESP8266-B、Arduino UNO、舵机外接电源）。
2. 等待两块ESP8266连接上Wi-Fi。
3. 手持遥控端，缓慢左右转动手腕 -> 机械臂底座跟随旋转；缓慢上下点头 -> 大臂跟随俯仰。
4. 若方向相反，修改`arduino_control.ino`中`map()`函数的输出顺序。

## 调试与优化
舵机抖动：检查外接电源是否足够（5V/2A以上），并确保所有GND共地。
WiFi连接失败：确认热点为2.4GHz频段。
运动不平滑：调整代码中的`DEAD_ZONE`、`MAX_RATE`、`FILTER_ALPHA`参数。
